# Eldersafe - IoT Monitoring System

A complete, distributed IoT system for monitoring elderly residents using ESP32-based sensors and Raspberry Pi edge servers.

## Features

- **Automatic WiFi Provisioning**: Auto-detect and configure new ESP32 devices
- **Real-time Telemetry**: Collect temperature, humidity, battery voltage, and uptime
- **Persistent Storage**: SQLite/MySQL database with 24/7 availability
- **RESTful API**: FastAPI backend for device management and data queries
- **Secure Connectivity**: TCP socket with MAC-based device authentication
- **Containerized Services**: Docker Compose for easy deployment
- **Scalable**: Support for 10-20 devices per Raspberry Pi

## System Requirements

### Raspberry Pi 
- **Model**: Raspberry Pi 4 (2GB+ RAM recommended)
- **OS**: Raspberry Pi OS (Lite or full)
- **Storage**: 16GB microSD card or larger
- **Network**: Ethernet + built-in WiFi
- **Power**: 5V 3A USB-C power supply

### ESP32
- **Board**: ESP32 DevKit, Heltec LoRa32, or similar
- **Storage**: 4MB SPIFFS or larger
- **Libraries**: WiFi (built-in), WebServer, Preferences, ArduinoJson v6+

## Installation

### 1. Flash ESP32 Firmware

1. Install Arduino IDE or PlatformIO
2. Install required libraries:
   - ArduinoJson (v6+)
3. Open `esp32/eldersafe_firmware.ino`
4. Configure board type: **ESP32 DevKit** (or your flavor)
5. Upload firmware to ESP32

### 2. Initialize Raspberry Pi

On a clean Raspberry Pi with Internet access:

```bash
# Clone repository
git clone https://github.com/yourorg/eldersafe.git
cd eldersafe

# Make scripts executable
chmod +x rpi/scripts/*.sh

# Run installation (ONE TIME ONLY)
sudo rpi/scripts/install.sh
```

The install script will:
- Install system dependencies (hostapd, dnsmasq, Docker, Docker Compose)
- Generate random WiFi password
- Create `/etc/eldersafe/.env` with credentials
- Configure systemd services
- Start all services

### 3. Boot the System

The Raspberry Pi should boot with these services active:

```bash
# Check service status
sudo systemctl status eldersafe-setup-network.service
sudo systemctl status eldersafe-docker.service
```

View logs:

```bash
# Network setup logs
sudo journalctl -u eldersafe-setup-network.service -f

# Docker service logs
sudo docker logs eldersafe-fastapi
sudo docker logs eldersafe-socket
sudo docker logs eldersafe-mysql
```

## Usage

### 1. Power On ESP32

When powered on without WiFi credentials (first boot):
- ESP32 enters **Setup Mode**
- Creates AP: `ELDERSAFE_SETUP_AABBCC` (last 6 MAC digits)
- Opens HTTP server on 192.168.4.1

### 2. Manual Provisioning

The RPI network is designed for stable connectivity. To provision a device:
- Connect your phone/laptop to the ESP32's `ELDERSAFE_SETUP_XXXX` AP
- Go to `http://192.168.4.1`
- Enter the RPi WiFi credentials displayed on your dashboard
- The ESP32 will reboot and connect permanently

Device is provisioned when:
- It connects to `ELDERSAFE_SECURE` WiFi
- Socket server authenticates its MAC
- First telemetry data arrives in the database

### 3. Monitor Devices via API

```bash
# List all registered devices
curl http://127.0.0.1:8000/api/iot-devices

# Register new device
curl -X POST http://127.0.0.1:8000/api/iot-devices/register \
  -H "Content-Type: application/json" \
  -d '{
    "mac_address": "AA:BB:CC:DD:EE:FF",
    "device_name": "Bedroom Sensor",
    "location": "Bedroom"
  }'

# Get latest telemetry
curl http://127.0.0.1:8000/api/iot-devices/1/telemetry/latest?limit=10
```

## Architecture

### Raspberry Pi (Host)
- **hostapd**: WiFi Access Point (SSID: `ELDERSAFE_SECURE`)
- **dnsmasq**: DHCP server + DNS (192.168.10.10-249)
- **setup_network.sh**: Network configuration (systemd-oneshot)

- **iptables**: NAT routing (Ethernet в†” WiFi)

### Docker Services
| Service | Port | Role |
|---------|------|------|
| MySQL 8.0 | 3306 | Data persistence |
| FastAPI | 8000 | REST API |
| Socket Server | 9000 | TCP listener for ESP32 |

## Configuration

Configuration is stored in `/etc/eldersafe/.env`:

```bash
# WiFi settings
WIFI_SSID=ELDERSAFE_SECURE
WIFI_PASSWORD=<auto-generated>

# Network
RPI_IP=192.168.10.1

# Database
MYSQL_ROOT_PASSWORD=<auto-generated>
MYSQL_USER=eldersafe
MYSQL_PASSWORD=<auto-generated>
MYSQL_DATABASE=eldersafe_db

# Services
SECRET_KEY=<auto-generated>
SOCKET_PORT=9000
```

To change WiFi password after installation:

```bash
# 1. Edit the .env file
sudo nano /etc/eldersafe/.env

# 2. Update hostapd config
sudo nano /etc/eldersafe/hostapd/hostapd.conf
# Change: wpa_passphrase=YOUR_NEW_PASSWORD

# 3. Restart hostapd
sudo killall hostapd dnsmasq
sudo /usr/sbin/hostapd -B /etc/eldersafe/hostapd/hostapd.conf
sudo /usr/sbin/dnsmasq -C /etc/eldersafe/dnsmasq/dnsmasq.conf
```

## Troubleshooting

### ESP32 Won't Provision

1. Ensure hostapd is running:
   ```bash
   ps aux | grep hostapd
   ```

4. Manually scan for ESP32:
   ```bash
   sudo iwlist wlan0 scan | grep ESSID
   ```

### Socket Connection Fails

1. Check socket server is listening:
   ```bash
   sudo netstat -tlnp | grep 9000
   ```

2. Check FastAPI is running:
   ```bash
   curl http://127.0.0.1:8000/health
   ```

3. Check device is registered:
   ```bash
   curl http://127.0.0.1:8000/api/iot-devices/check-mac/AABBCCDDEEFF
   ```

### No Telemetry Data

1. Check MySQL:
   ```bash
   docker exec eldersafe-mysql mysql -u root -p$MYSQL_ROOT_PASSWORD -e "SELECT * FROM eldersafe_db.telemetry_data LIMIT 5;"
   ```

2. Check socket server logs:
   ```bash
   docker logs eldersafe-socket -f
   ```

3. Check device is active and connected:
   ```bash
   curl http://127.0.0.1:8000/api/iot-devices
   ```

### Making Changes

1. **ESP32 Firmware**: Edit `esp32/eldersafe_firmware.ino`, recompile and upload
2. **Network Scripts**: Edit `rpi/scripts/*.sh`, redeploy with `sudo systemctl restart eldersafe-setup-network`
3. **FastAPI**: Edit `rpi/docker/api/*.py`, rebuild: `docker-compose build fastapi && docker-compose up -d`
4. **Socket Server**: Edit `rpi/docker/socket_server/socket_server.py`, rebuild: `docker-compose build socket-server && docker-compose up -d`
